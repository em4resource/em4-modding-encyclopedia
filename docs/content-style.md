# Content Style Guide

## Article Structure

Each article should answer these questions:

1. What is the feature or concept?
2. Where does it live in the mod folder?
3. What files does it depend on?
4. What can go wrong?
5. How do you test it?

## Good EM4 Documentation Habits

- Use real file paths when useful.
- Name required virtual objects exactly.
- List command names exactly as they appear in scripts.
- Separate person commands, vehicle commands, object commands, and dummy commands.
- Avoid vague instructions like "add the script." Say which prototype gets which command.
- Include warnings for parser quirks, DDS icon issues, and command menu conflicts.
